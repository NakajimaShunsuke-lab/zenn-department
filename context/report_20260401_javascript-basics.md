## 改善点・知見

- **初心者向け記事の分量**: 5,000〜7,000字が挫折しにくい適切な分量。既存のZenn書籍（数万字〜20万字超）とは異なるアプローチで差別化できた
- **つまずきポイントの独立セクション化**: `==`vs`===`、暗黙の型変換、`var`問題を独立セクションにしたことで、FAQ的な使い方もできる構成になった
- **テンプレートリテラルの重要性**: 2026年のJS記事では`+`結合ではなくテンプレートリテラルが標準。初心者向けでも最初からモダンな書き方を紹介すべき
- **Mermaid図の活用**: 「Webの3つの柱（HTML/CSS/JS）」の図は初心者の理解を助ける効果が高い。初心者記事では概念図を積極的に入れるとよい
