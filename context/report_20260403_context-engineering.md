## 主な判断・修正履歴

- **企画段階**: 既存のZenn/Qiita記事は概念紹介が中心のため、「具体的な設定ファイル例を含み、読後すぐ実践できる」構成で差別化。4つのコアテクニック（Writing Out / Selection / Compression / Segmentation）をフレームワークとして体系化する方針を決定
- **企画段階**: 中級者向けのため、LLMの基本的な仕組みの説明は省略し、実践的な内容に重点を置く方針を決定
- **執筆段階**: 各コアテクニックを「課題→解決策→具体例」の統一フォーマットで整理。Compressionには「悪い例 vs 良い例」のコード比較を挿入
- **執筆段階**: プロジェクト設定ファイルのセクションでは、3ツール（Claude Code / Cursor / GitHub Copilot）の実用的な設定例と比較表を作成
- **レビュー段階**: Mermaid `block-beta` 記法をZenn互換性のある `graph TD` に変更
- **レビュー段階**: topicsから記事との関連が薄い「ChatGPT」を削除し、主題の「コンテキストエンジニアリング」に変更
- **レビュー段階**: 冒頭の文体を「です/ます」調に統一

## 改善点・知見

- **中級者向け記事の構成**: 基本説明を省略し、フレームワーク提示→具体例→実践という流れが効果的。初心者向けとは明確に異なるアプローチが必要
- **「課題→解決策→具体例」パターンが有効**: 各テクニックを統一フォーマットで整理したことで、読者が体系的に理解しやすい構成になった。今後の解説系記事でも活用できるパターン
- **ツール比較表の効果**: CLAUDE.md / Cursor Rules / copilot-instructions.md の比較表は、読者が自分の環境に合った選択をしやすくする。複数ツールを扱う記事では比較表を積極的に活用すべき
- **Mermaid記法の互換性に注意**: `block-beta` 等の新しい記法はZennでサポートされない場合がある。`graph TD`/`graph LR` 等の安定した記法を優先すべき
- **topicsには記事の主題キーワードを必ず含める**: 当たり前だが、執筆時にtopicsを仮設定したまま見落とすケースがある。レビューチェック項目として明示的に確認する価値がある
- **OGP画像プロンプトは英語で作成**: 過去レポートの知見どおり英語プロンプトをデフォルトで使用。今後も継続
