# 編集部Zenn部門 — Agent Teams 設立プロンプト

Zenn技術記事の企画・執筆・レビュー・公開を行うAIエージェントチームです。
記事の管理は `zenn-repo/` で Zenn CLI を使用して行います。

### 1. AIエージェントの役割分担

技術記事は正確性と構成が命となるため、執筆プロセスを専門領域ごとにAIエージェントへ分割しています。
各エージェントはスラッシュコマンド（Skill）として実装済みです。

| エージェント | コマンド | 役割 |
|---|---|---|
| リサーチャーAI | `/zenn-director` | 企画・情報収集・構成案作成 |
| ライターAI | `/zenn-write` | Markdown執筆・サンプルコード作成 |
| レビュアー・校正AI | `/zenn-review` | 品質管理・テクニカルチェック |
| 画像生成プロンプトAI | `/zenn-visual` | アイキャッチ・図解用プロンプト作成 |
| 公開オペレーション | `/zenn-ops` | 公開前チェック・published切り替え |
| レポートAI | `/zenn-report` | 記事作成レポートの作成 |
| ナレッジAI | `/zenn-knowledge` | レポート＋アナリティクスからナレッジ管理 |
| 統括エージェント | `/zenn-full` | 全フェーズ一括実行 |

#### 各エージェントの詳細

* **リサーチャーAI (`/zenn-director`)**
    * **役割:** 指定された技術テーマに基づく最新情報の収集、公式ドキュメントの読み込み、競合記事の調査を行います。`context/knowledge/` のナレッジも参照します。
    * **出力:** 企画書（`zenn-repo/briefs/` に配置）。見出し構成案、参考URLリスト、サンプルコード要件などをまとめます。
* **ライターAI (`/zenn-write`)**
    * **役割:** リサーチャーが作成した構成案をもとに、`CONTRIBUTING_ZENN.md` のガイドラインに従い、Zennに適した論理的でわかりやすい文章をMarkdown形式で執筆します。
    * **出力:** 記事の初稿（`zenn-repo/articles/` に配置、`published: false`）。必要に応じてサンプルコードの実装と解説文も生成します。
* **レビュアー・校正AI (`/zenn-review`)**
    * **役割:** 初稿の誤字脱字、論理の飛躍、コードの不備をチェックし、Zennの読者層に合ったトーン＆マナー（技術的な正確性、客観性など）になっているかを審査します。
    * **出力:** 修正提案のフィードバック、または自動修正済みの第2稿。
* **画像生成プロンプトAI (`/zenn-visual`)**
    * **役割:** 記事の内容に合ったアイキャッチ画像や図解用画像を生成するためのプロンプトを作成します。実際の画像生成は人間がGeminiのNanobananaを使用して行います。
    * **出力:** 画像生成用のプロンプト（画像の構図、スタイル、要素などを指定した詳細な指示文）。画像ファイルは `zenn-repo/images/{slug}/` に配置します。
* **公開オペレーション (`/zenn-ops`)**
    * **役割:** 公開前チェックリスト（frontmatter、コンテンツ、品質）の確認を行い、ユーザー承認後に `published: true` へ切り替えます。
    * **出力:** チェック結果レポート、および承認後の公開済み記事。コミットメッセージは `feat(article): {記事タイトルの要約}`。
* **レポートAI (`/zenn-report`)**
    * **役割:** 記事作成プロセス全体の進捗や各AIの出力結果をまとめ、記事作成におけるレポートを作成します。
    * **出力:** 記事作成の工程・判断・修正履歴などをまとめたレポート。`context/report/report_YYYYMMDD_テーマ名.md` の形式で保存し、`npm run sync-notion` でNotionデータベースに同期します。
* **ナレッジAI (`/zenn-knowledge`)**
    * **役割:** `context/report/` のレポートと `context/analytics/` のGoogle Analyticsデータを分析し、ナレッジを作成・更新します。処理済みファイルは `context/old_report/` にアーカイブします。
    * **出力:** `context/knowledge/` にナレッジファイルを作成・更新。今後の記事執筆における改善点・知見として各エージェントが参照します。

### 2. 人間（管理者）の役割

AIチームを機能させるための、人間側の重要な役割です。

* **統括ディレクター（企画とプロンプト管理）:**
    * 記事のテーマ選定、ターゲット読者の設定、および各AIエージェントへのプロンプト（指示内容）の継続的なチューニングを行います。
* **最終承認と運用・エンゲージメント対応:**
    * AIが作成した最終原稿のファクトチェックを行い、Zennへの公開作業を承認します。公開後の読者からのコメント対応やSNS等でのエンゲージメント対応はAIに任せず、人間が直接引き受けることで、読者との適切なコミュニケーションを担保します。

### 3. ワークフロー

記事作成は以下のフェーズで進行します。各フェーズの完了後、人間（統括ディレクター）が確認・承認を行い次のフェーズへ進みます。
全フェーズを一括実行する場合は `/zenn-full` を使用します（各フェーズ完了時にユーザー確認あり）。

#### フェーズ1: 企画・テーマ設定（人間）
1. 統括ディレクターが記事のテーマ・ターゲット読者・ゴールを決定する
2. `/zenn-director` を実行する

#### フェーズ2: リサーチ・構成案作成（`/zenn-director`）
1. `context/knowledge/` のナレッジを参照し、指定テーマの情報を収集する
2. 見出し構成案・参考URLリスト・サンプルコード要件をまとめ、企画書を `zenn-repo/briefs/` に作成する
3. **人間が構成案を確認・承認する**

#### フェーズ3: 執筆（`/zenn-write`）
1. 承認済みの構成案をもとに `CONTRIBUTING_ZENN.md` に従いMarkdown形式で初稿を執筆する
2. 記事ファイルを `zenn-repo/articles/` に配置する（`published: false`）
3. **人間が初稿の方向性を確認する**

#### フェーズ4: レビュー・校正（`/zenn-review`）
1. 初稿の誤字脱字・論理の飛躍・コードの不備をチェックする
2. Zennの読者層に合ったトーン＆マナーになっているか審査する
3. 修正提案のフィードバックまたは修正済みの第2稿を出力する
4. **人間が修正内容を確認し、必要に応じてライターAIへ差し戻す**

#### フェーズ5: 画像作成（`/zenn-visual` → 人間）
1. 記事の内容に合ったアイキャッチ画像・図解用の画像生成プロンプトを作成する
2. **人間がGeminiのNanobananaを使用して画像を生成する**
3. 生成された画像を `zenn-repo/images/{slug}/` に配置し記事に挿入する

#### フェーズ6: 公開前チェック・公開（`/zenn-ops`）
1. frontmatter・コンテンツ・品質のチェックリストを確認する
2. チェック結果をユーザーに報告する
3. **人間の承認を得てから `published: true` に変更する**
4. コミットメッセージ `feat(article): {記事タイトルの要約}` でコミットする
5. mainブランチへのpushでZennに自動反映される

#### フェーズ7: レポート作成（`/zenn-report`）
1. 記事作成プロセス全体の工程・判断・修正履歴をまとめたレポートを作成する
2. `context/report/report_YYYYMMDD_テーマ名.md` の形式で保存する
3. `npm run sync-notion` でNotionデータベースに同期する
4. 過去のレポートを参照し、次回以降の記事執筆に活かせる改善点・知見を抽出する

#### フェーズ8: 公開後対応（人間）
1. 読者からのコメント対応を行う
2. SNS等でのエンゲージメント対応を行う
3. 必要に応じて記事の追記・修正を行う

#### ナレッジ更新（`/zenn-knowledge` — 随時）
1. `context/report/` のレポートと `context/analytics/` のアナリティクスデータを分析する
2. `context/knowledge/` にナレッジを作成・更新する
3. 処理済みのレポート・アナリティクスは `context/old_report/` にアーカイブする
4. `npm run sync-notion:update` でアナリティクスデータを含めたNotion更新を行う

```
【ワークフロー全体図】

人間：企画・テーマ設定
        ↓
/zenn-director：情報収集・構成案作成 → briefs/
        ↓ ←── 人間：構成案の確認・承認
/zenn-write：Markdown執筆 → articles/（published: false）
        ↓ ←── 人間：初稿の方向性確認
/zenn-review：品質管理・テクニカルチェック
        ↓ ←── 人間：修正内容の確認（差し戻しの場合は /zenn-write へ）
/zenn-visual：画像生成プロンプト作成
        ↓
人間：Nanobananaで画像生成 → images/{slug}/
        ↓
/zenn-ops：公開前チェック → 人間承認 → published: true
        ↓
/zenn-report：レポート作成 → context/report/ → Notion同期
        ↓
人間：公開後対応（コメント・エンゲージメント）

【一括実行: /zenn-full（各フェーズ完了時にユーザー確認あり）】

【随時: /zenn-knowledge でレポート＋アナリティクス → ナレッジ更新】
```

### 4. ディレクトリ構成

```
zenn-repo/
├── articles/          # 記事ファイル（{slug}.md）
├── briefs/            # 企画書（{テーマ名}.md）
├── images/{slug}/     # 画像関連ファイル
└── .templates/        # 記事・企画書テンプレート

context/
├── report/            # 記事作成レポート（未処理）
├── analytics/         # Google Analytics CSVエクスポート（計測期間: YYYYMMDD-YYYYMMDD.csv）
├── knowledge/         # ナレッジ集（蓄積）
└── old_report/        # /zenn-knowledge で処理済みのアーカイブ

scripts/               # 自動化スクリプト（Notion同期、GA取得等）
```

### 5. 外部連携

* **Notion同期:** レポート作成後に `npm run sync-notion` で自動同期。アナリティクスデータ含む更新は `npm run sync-notion:update`。
* **Google Analytics:** `npm run fetch-analytics` でCSVエクスポートを `context/analytics/` に取得。週次取得は `npm run fetch-analytics:weekly`。
* **Zennデプロイ:** mainブランチへのpushでZennに自動反映（GitHub連携）。

### 6. 運用ルール

* **Zenn用ガイドラインの徹底:** `CONTRIBUTING_ZENN.md` にZennで好まれる文体（結論ファースト、簡潔な表現、過剰な装飾を避けるなど）を定義。全エージェントがこのガイドラインを遵守します。
* **ワークフローの自動化:** 各フェーズはスラッシュコマンドで実行可能。`/zenn-full` で全フェーズを一括実行できます。
* **ナレッジの継続的改善:** レポートとアナリティクスから `/zenn-knowledge` でナレッジを定期的に更新し、記事品質の向上に活かします。
