# 編集部Zenn部門の設立

Agent Teamsとして設立したい
Zennの記事の管理はzenn-repoで行いCLIで行います
このAgent Teamsの仕様の詳細をREADEMEに記載してください

### 1. AIエージェントの役割分担

技術記事は正確性と構成が命となるため、執筆プロセスを3つのAIに分割することをおすすめします。

* **リサーチャーAI (情報収集・構成案作成)**
    * **役割:** 指定された技術テーマに基づく最新情報の収集、公式ドキュメントの読み込み、競合記事の調査を行います。
    * **出力:** 記事の骨組みとなる見出し構成案、参考URLリスト、必要なサンプルコードの要件などをまとめます。
* **ライターAI (Markdown執筆・コーディング)**
    * **役割:** リサーチャーが作成した構成案をもとに、Zennに適した論理的でわかりやすい文章をMarkdown形式で執筆します。
    * **出力:** 記事の初稿。必要に応じてサンプルコードの実装と解説文も生成します。
* **レビュアー・校正AI (品質管理・テクニカルチェック)**
    * **役割:** 初稿の誤字脱字、論理の飛躍、コードの不備をチェックし、Zennの読者層に合ったトーン＆マナー（技術的な正確性、客観性など）になっているかを審査します。
    * **出力:** 修正提案のフィードバック、または自動修正済みの第2稿。
* **画像生成プロンプトAI (アイキャッチ・図解用画像の生成支援)**
    * **役割:** 記事の内容に合ったアイキャッチ画像や図解用画像を生成するためのプロンプトを作成します。実際の画像生成は人間がGeminiのNanobananaを使用して行います。
    * **出力:** 画像生成用のプロンプト（画像の構図、スタイル、要素などを指定した詳細な指示文）。
* **レポートAI (記事作成レポートの作成)**
    * **役割:** 記事作成プロセス全体の進捗や各AIの出力結果をまとめ、記事作成におけるレポートを作成します。
    * **出力:** 記事作成の工程・判断・修正履歴などをまとめたレポート。`context/` ディレクトリに保存し、今後の記事執筆におけるナレッジとして蓄積・活用します。

### 2. 人間（管理者）の役割

AIチームを機能させるための、人間側の重要な役割です。

* **統括ディレクター（企画とプロンプト管理）:**
    * 記事のテーマ選定、ターゲット読者の設定、および各AIエージェントへのプロンプト（指示内容）の継続的なチューニングを行います。
* **最終承認と運用・エンゲージメント対応:**
    * AIが作成した最終原稿のファクトチェックを行い、Zennへの公開作業を行います。公開後の読者からのコメント対応やSNS等でのエンゲージメント対応はAIに任せず、人間が直接引き受けることで、読者との適切なコミュニケーションを担保します。


### 3. ワークフロー

記事作成は以下のフェーズで進行します。各フェーズの完了後、人間（統括ディレクター）が確認・承認を行い次のフェーズへ進みます。

#### フェーズ1: 企画・テーマ設定（人間）
1. 統括ディレクターが記事のテーマ・ターゲット読者・ゴールを決定する
2. 各AIエージェントへの指示（プロンプト）を準備する

#### フェーズ2: リサーチ（リサーチャーAI）
1. 指定テーマに基づき最新情報・公式ドキュメント・競合記事を調査する
2. 見出し構成案・参考URLリスト・サンプルコード要件をまとめる
3. **人間が構成案を確認・承認する**

#### フェーズ3: 執筆（ライターAI）
1. 承認済みの構成案をもとにMarkdown形式で初稿を執筆する
2. 必要に応じてサンプルコードの実装と解説文を作成する
3. **人間が初稿の方向性を確認する**

#### フェーズ4: レビュー・校正（レビュアー・校正AI）
1. 初稿の誤字脱字・論理の飛躍・コードの不備をチェックする
2. Zennの読者層に合ったトーン＆マナーになっているか審査する
3. 修正提案のフィードバックまたは修正済みの第2稿を出力する
4. **人間が修正内容を確認し、必要に応じてライターAIへ差し戻す**

#### フェーズ5: 画像作成（画像生成プロンプトAI → 人間）
1. 記事の内容に合ったアイキャッチ画像・図解用の画像生成プロンプトを作成する
2. **人間がGeminiのNanobananaを使用して画像を生成する**
3. 生成された画像を記事に挿入する

#### フェーズ6: 最終承認・公開（人間）
1. 人間が最終原稿のファクトチェックを行う
2. 画像・構成・文章の最終確認を行う
3. Zennへ公開する

#### フェーズ7: レポート作成（レポートAI）
1. 記事作成プロセス全体の工程・判断・修正履歴をまとめたレポートを作成する
2. 作成したレポートを `context/` ディレクトリに保存する（例: `context/report_YYYYMMDD_テーマ名.md`）
3. 過去のレポートを参照し、次回以降の記事執筆に活かせる改善点・知見を抽出する
4. `context/` に蓄積されたレポートは、リサーチャーAI・ライターAI・レビュアーAIが今後の記事作成時にナレッジとして参照する

#### フェーズ8: 公開後対応（人間）
1. 読者からのコメント対応を行う
2. SNS等でのエンゲージメント対応を行う
3. 必要に応じて記事の追記・修正を行う

```
【ワークフロー全体図】

人間：企画・テーマ設定
        ↓
リサーチャーAI：情報収集・構成案作成
        ↓ ←── 人間：構成案の確認・承認
ライターAI：Markdown執筆・コーディング
        ↓ ←── 人間：初稿の方向性確認
レビュアー・校正AI：品質管理・テクニカルチェック
        ↓ ←── 人間：修正内容の確認（差し戻しの場合はライターAIへ）
画像生成プロンプトAI：プロンプト作成
        ↓
人間：Nanobananaで画像生成 → 記事に挿入
        ↓
人間：最終承認 → Zennへ公開
        ↓
レポートAI：記事作成レポート作成
        ↓
人間：公開後対応（コメント・エンゲージメント）
```

### 4. 運用を軌道に乗せるポイント

* **Zenn用ガイドラインの徹底:** AIへのシステムプロンプトに、「Zennで好まれる文体（結論ファースト、簡潔な表現、過剰な装飾を避けるなど）」を厳格に設定します。
ガイドラインは"CONTRIBUTING_ZENN.md"に記載しています
* **ワークフローの自動化:** claude codeの
コマンドで全フェーズを一括で対応できるようにしてください
