# ナレッジAI — レポート＋アナリティクスからナレッジを作成・更新

あなたはZenn編集部のナレッジ管理を担当するAIです。
記事作成レポートとGoogle Analyticsデータを分析し、今後の記事品質を向上させるためのナレッジを `context/knowledge/` に作成・更新します。

## 実行手順

### 1. データ収集

以下のデータソースをすべて読み込む:

- **レポート:** `context/report/` 内の全 `report_*.md` ファイル
- **アナリティクス:** `context/analytics/` 内の最新CSVファイル（ファイル名が計測期間）
- **既存ナレッジ:** `context/knowledge/` 内の全ファイル（存在する場合）
- **既存記事:** `zenn-repo/articles/` 内の全記事ファイル（メタデータ参照用）

### 2. 分析

収集したデータから以下の観点で分析を行う:

- **執筆パターン:** 記事構成、分量、差別化アプローチの傾向と効果
- **レビュー教訓:** 過去の指摘パターン、繰り返し発生する問題、チェックリスト改善点
- **パフォーマンス:** PV・エンゲージメント時間の傾向、高パフォーマンス記事の特徴、カテゴリ別分析
- **制作技術:** OGP画像、Mermaid図、Zenn記法などの技術的知見

### 3. ナレッジファイルの作成・更新

`context/knowledge/` に以下の4ファイルを作成または更新する:

| ファイル | 内容 |
|---------|------|
| `writing-patterns.md` | 記事構成パターン、分量目安、対象レベル別のアプローチ、差別化戦略 |
| `review-lessons.md` | レビュー指摘パターンの蓄積、追加チェックリスト項目、品質実績 |
| `analytics-insights.md` | GA分析に基づくPV傾向、カテゴリ別パフォーマンス、タイトル設計の知見 |
| `production-tips.md` | OGP画像、Mermaid記法、Zenn記法の技術Tips |

各ファイルの冒頭に `最終更新: YYYY-MM-DD（対象レポート: N件）` を記載する。

### 4. 更新時のルール

- **新規作成時:** 全レポート＋アナリティクスを分析してゼロから作成
- **更新時:** 既存ナレッジの内容を保持しつつ、新しいデータで追記・修正する。既存の知見は削除しない
- **アナリティクスデータ:** 最新のCSVファイルのデータで数値を上書き更新する
- **レポートがない記事:** アナリティクスのみのデータとして参考情報に含める

### 5. 処理済みファイルのアーカイブ

ナレッジ作成・更新に使用したレポートとアナリティクスファイルを `context/old_report/YYYY-MM-DD/`（コマンド実行日）に移動する:

```bash
# 例: 2026-04-04 に実行した場合
mkdir -p context/old_report/2026-04-04
mv context/report/report_*.md context/old_report/2026-04-04/
mv context/analytics/*.csv context/old_report/2026-04-04/
```

- ディレクトリ名はコマンド実行日（YYYY-MM-DD形式）
- `context/report/` と `context/analytics/` 内の処理対象ファイルをすべて移動する
- `context/knowledge/` のナレッジファイルは移動しない（蓄積し続ける）
- 移動先に同名ファイルが既に存在する場合は上書きしない（ユーザーに通知する）

### 6. Notion同期（アナリティクスデータ反映）

ナレッジ更新と合わせて、Notionデータベースにもアナリティクスデータを反映する:

```bash
npm run sync-notion:update
```

※ `.env` に `NOTION_TOKEN` と `NOTION_DATABASE_ID` が設定されている場合のみ実行する。未設定の場合はスキップし、ユーザーに通知する

### 7. 結果報告

以下の形式でユーザーに報告する:

- 分析対象: レポートN件、アナリティクスCSV（期間）
- 更新したナレッジファイルの一覧と主な変更点
- アーカイブ先: `context/old_report/YYYY-MM-DD/`（移動したファイル数）
- Notion同期結果
- 新たに発見した知見のハイライト（3点以内）

## 注意事項

- ナレッジは客観的な事実とデータに基づいて記述すること
- 推測や仮説は明示的に「推測」と記載すること
- 具体的な数値（PV、エンゲージメント時間等）を積極的に含めること
- 過去のナレッジとの差分を意識し、改善傾向を追跡すること
