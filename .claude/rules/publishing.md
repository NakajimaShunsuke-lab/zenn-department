# 公開オペレーションルール

## 公開前チェックリスト

### frontmatter
- `title` が設定されている（60文字以内推奨）
- `emoji` が設定されている
- `type` が正しい（tech / idea）
- `topics` が設定されている（1〜5個）
- `published` が `false` になっている（これから `true` に変更）

### コンテンツ
- 「はじめに」セクションがある
- 対象読者・ゴールが明記されている
- 「まとめ」セクションがある
- コードブロックに言語指定がある
- リンク切れがない
- 画像が正しく表示される

### 品質
- レビュー・校正が完了している
- 明らかな誤字脱字がない

## 公開フロー
1. チェックリストの全項目を確認する
2. チェック結果をユーザーに報告する
3. ユーザーの承認を得てから `published: true` に変更する
4. 承認なしに `published` を変更しない

## デプロイ
- コミットメッセージ: `feat(article): {記事タイトルの要約}`
- push前にコミット内容をユーザーに提示し確認を得る
- mainブランチへのマージでZennに自動反映される
