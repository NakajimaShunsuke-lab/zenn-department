# 編集部 Zenn部門 — Agent Teams

Claude CodeのカスタムスラッシュコマンドによるAIエージェントチームで、Zenn技術記事の企画から公開までを一気通貫で支援します。

## チーム構成

| エージェント | スラッシュコマンド | 役割 |
|---|---|---|
| リサーチャーAI | `/zenn-director` | 情報収集・構成案作成・企画書の策定 |
| ライターAI | `/zenn-write` | Markdown執筆・サンプルコード作成 |
| レビュアー・校正AI | `/zenn-review` | 品質管理・テクニカルチェック |
| 画像生成プロンプトAI | `/zenn-visual` | アイキャッチ・図解用の画像生成プロンプト作成 |
| レポートAI | `/zenn-report` | 記事作成プロセスのレポート作成・ナレッジ蓄積 |
| 公開オペレーション | `/zenn-ops` | 公開前チェック・published切り替え |
| **全フェーズ一括実行** | **`/zenn-full`** | **企画→執筆→レビュー→画像→公開→レポートを通しで実行** |

## 各エージェントの詳細

### リサーチャーAI（`/zenn-director`）

指定された技術テーマに基づき、最新情報・公式ドキュメント・競合記事を調査し、記事の骨組みとなる企画書を作成します。

- **入力:** テーマ・ターゲット読者・ゴール
- **出力:** `briefs/{テーマ名}.md`（見出し構成案・参考URLリスト・サンプルコード要件）

### ライターAI（`/zenn-write`）

承認済みの企画書をもとに、CONTRIBUTING_ZENN.md のガイドラインに従い、Zennに適した技術記事をMarkdown形式で執筆します。

- **入力:** `briefs/{テーマ名}.md`
- **出力:** `articles/{slug}.md`（published: false）

### レビュアー・校正AI（`/zenn-review`）

初稿の誤字脱字・論理の飛躍・コードの不備をチェックし、Zennの読者層に合ったトーン＆マナーになっているかを審査します。

- **入力:** `articles/{slug}.md`
- **出力:** レビュー結果のフィードバック、または修正済み記事

### 画像生成プロンプトAI（`/zenn-visual`）

記事内容に合ったアイキャッチ画像や図解用画像を生成するためのプロンプトを作成します。実際の画像生成は人間がGeminiのNanobananaを使用して行います。

- **入力:** 記事の内容
- **出力:** `images/{slug}/ogp-design.md`（画像生成用プロンプト）、Mermaid図解

### レポートAI（`/zenn-report`）

記事作成プロセス全体の進捗や各AIの出力結果をまとめ、ナレッジとして蓄積します。

- **入力:** 記事作成プロセスの成果物一式
- **出力:** `context/report_YYYYMMDD_テーマ名.md`

### 公開オペレーション（`/zenn-ops`）

公開前の最終チェックリストを実施し、記事を公開状態に切り替えます。

- **入力:** レビュー・画像作成完了済みの記事
- **出力:** `published: true` に変更された記事

## ワークフロー

```
【ワークフロー全体図】

人間：企画・テーマ設定
        ↓
リサーチャーAI：/zenn-director（情報収集・構成案作成）
        ↓ ←── 人間：構成案の確認・承認
ライターAI：/zenn-write（Markdown執筆・コーディング）
        ↓ ←── 人間：初稿の方向性確認
レビュアー・校正AI：/zenn-review（品質管理・テクニカルチェック）
        ↓ ←── 人間：修正内容の確認（差し戻しの場合はライターAIへ）
画像生成プロンプトAI：/zenn-visual（プロンプト作成）
        ↓
人間：Nanobananaで画像生成 → 記事に挿入
        ↓
人間：最終承認 → /zenn-ops で公開
        ↓
レポートAI：/zenn-report（記事作成レポート作成）
        ↓
人間：公開後対応（コメント・エンゲージメント）
```

### フェーズ詳細

| フェーズ | 担当 | 内容 | 成果物 |
|---|---|---|---|
| 1. 企画・テーマ設定 | 人間 | テーマ・ターゲット・ゴール決定 | プロンプト入力 |
| 2. リサーチ | `/zenn-director` | 情報収集・構成案作成 | `briefs/{テーマ名}.md` |
| 3. 執筆 | `/zenn-write` | Markdown執筆・コード作成 | `articles/{slug}.md` |
| 4. レビュー・校正 | `/zenn-review` | 品質チェック・修正 | 修正済み記事 |
| 5. 画像作成 | `/zenn-visual` + 人間 | プロンプト作成 → Nanobananaで生成 | OGP画像・図解 |
| 6. 最終承認・公開 | `/zenn-ops` + 人間 | チェックリスト → 公開 | 公開済み記事 |
| 7. レポート | `/zenn-report` | プロセスレポート作成 | `context/report_*.md` |
| 8. 公開後対応 | 人間 | コメント・エンゲージメント対応 | — |

## 人間（管理者）の役割

- **統括ディレクター（企画とプロンプト管理）:** テーマ選定、ターゲット読者の設定、各AIエージェントへのプロンプトのチューニング
- **最終承認と運用:** ファクトチェック、Zennへの公開、読者コメント・SNSエンゲージメント対応

## 一括実行

全フェーズを通しで実行する場合:

```
/zenn-full テーマ名
```

各フェーズ完了時にユーザーの確認・承認を挟みながら、企画からレポート作成まで一気通貫で進行します。

## ショートカット

個別コマンドで部分的に実行することも可能です:

- **急ぎの記事:** `/zenn-write` → `/zenn-review` → `/zenn-ops`
- **図解メインの記事:** `/zenn-director` → `/zenn-write` → `/zenn-visual` → `/zenn-ops`
- **軽い記事:** `/zenn-write` → `/zenn-ops`

## ディレクトリ構成

```
Zenn/
├── CLAUDE.md                  # エージェント共通指示
├── CONTRIBUTING_ZENN.md       # Zenn記事ガイドライン
├── README.md                  # 本ファイル（仕様書）
├── .claude/
│   ├── settings.json          # Claude Code設定
│   └── commands/              # カスタムスラッシュコマンド
│       ├── zenn-director.md   # リサーチャーAI
│       ├── zenn-write.md      # ライターAI
│       ├── zenn-review.md     # レビュアー・校正AI
│       ├── zenn-visual.md     # 画像生成プロンプトAI
│       ├── zenn-report.md     # レポートAI
│       ├── zenn-ops.md        # 公開オペレーション
│       └── zenn-full.md       # 全フェーズ一括実行
├── context/                   # レポート・ナレッジ蓄積
├── prompt/                    # 設立資料・プロンプト管理
└── zenn-repo/                 # Zenn CLIリポジトリ（記事管理）
    ├── articles/              # 記事本体
    ├── books/                 # 本（未使用）
    ├── briefs/                # 企画書
    ├── images/                # 画像
    ├── .templates/            # テンプレート
    └── .workflow/             # ワークフロー定義
```

## 運用ポイント

- **ガイドライン:** CONTRIBUTING_ZENN.md に定義されたZenn向け文体ルールを全エージェントが遵守
- **ナレッジ蓄積:** `context/` に蓄積されたレポートを各AIが参照し、記事品質を継続的に改善
- **Zenn CLI連携:** `zenn-repo/` ディレクトリでZenn CLIを使用し、プレビュー・公開を管理
